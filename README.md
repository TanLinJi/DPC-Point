# DPC-Point

Distribution-Constrained Prototype Caching for Point Cloud Test-Time Adaptation

![DPC-Point Architecture](assets/Architecture.png)

## Overview

DPC-Point is a training-free test-time adaptation method for 3D vision-language foundation models. During inference, the backbone model is frozen. DPC-Point calibrates predictions for sequentially arriving test point clouds with hierarchical prototype caches, a textual semantic distribution, an online visual distribution, and a visual-semantic distribution consistency score.

The current release supports three backbones:

- ULIP
- OpenShape
- Uni3D

The current release evaluates four datasets:

- ModelNet
- ModelNet-C
- ScanObjectNN
- ScanObjectNN-C

Distribution shifts in test point clouds can disrupt the alignment between visual features and class semantics in 3D vision-language foundation models, degrading zero-shot inference performance. Test-time adaptation (TTA) methods commonly build historical caches from sequentially arriving test samples and use them to refine predictions. However, most existing methods rely solely on low prediction entropy to identify high-confidence samples. Under complex distribution shifts, misclassified yet highly confident samples may therefore be admitted to the caches, resulting in cache contamination. To address this issue, DPC-Point (Distribution-Constrained Prototype Caching for Point Cloud Test-Time Adaptation), a prototype caching method constrained by probability distributions, is proposed. First, a hierarchical architecture comprising multiple complementary caches constructs reliable prototypes from representative features while preserving class-confusion information from uncertain samples. Next, a textual semantic distribution and an online visual distribution are modeled as two complementary Gaussian distributions. The former is estimated from manually designed class templates and supplementary class descriptions generated by a large language model, providing stable semantic class priors for zero-shot point cloud classification under distribution shifts. The latter is estimated from historical point cloud prototypes admitted to the hierarchical caches. It captures the overall distribution and evolution of class-specific features in the test stream, providing visual evidence for identifying more reliable prototypes. Finally, a visual-semantic distribution consistency score derived from the two distributions is combined with prediction entropy to constrain prototype cache updates. This allows the class-level information captured by the visual-semantic distributions and the historical features retained in the reliable prototype caches to jointly refine predictions for subsequent samples, thereby reducing cache contamination and improving overall performance under complex distribution shifts. Experiments on multiple datasets demonstrate that DPC-Point effectively improves the zero-shot inference performance of several 3D vision-language foundation models under geometric perturbations and distribution shifts in real-world scans, confirming its effectiveness and robustness.

## Reproduction

All commands below should be executed from the project root.

The recommended full reproduction workflow is:

```bash
conda env create -f environment.yml
conda activate dpcp

bash scripts/download/download_data_all.sh

python weights/download_openshape_weights.py --variant vitg14
python weights/download_uni3d_weights.py --with-task-ckpts

# Download ULIP weights from the official link below and place them under weights/ulip/.

bash scripts/zero_shot/run_all.sh 0
bash scripts/dpc_point/run_all.sh 0
```

### 1. Environment

We recommend creating the full Conda environment:

```bash
conda env create -f environment.yml
conda activate dpcp
```

If you already have a compatible Python, CUDA, and PyTorch environment, you can install only the pip dependencies:

```bash
conda create -n dpcp python=3.9 -y
conda activate dpcp
pip install -r requirements.txt
```

Main environment versions:

- Python 3.9
- PyTorch 1.12.0
- CUDA 11.6
- torchvision 0.13.0
- timm 0.9.16
- open-clip-torch 2.24.0

### 2. Datasets

DPC-Point uses only ModelNet, ModelNet-C, ScanObjectNN, and ScanObjectNN-C. The clean split is stored together with the corresponding corrupted dataset: ModelNet uses `data/modelnet_c/clean.h5`, and ScanObjectNN uses `data/sonn_c/hardest/clean.h5`.

Official dataset links and local download scripts:

| Dataset | Official Link | Download Script |
| --- | --- | --- |
| ModelNet / ModelNet-C | [Point-PRC modelnet_c](https://huggingface.co/datasets/auniquesun/Point-PRC/tree/main/new-3ddg-benchmarks/xset/corruption/modelnet_c) | `python scripts/download/download_data_modelnet_c.py` |
| ScanObjectNN / ScanObjectNN-C | [Point-PRC sonn_c](https://huggingface.co/datasets/auniquesun/Point-PRC/tree/main/new-3ddg-benchmarks/xset/corruption/sonn_c) | `python scripts/download/download_data_scanobjectnn_c.py` |

Download all datasets required by the released experiments:

```bash
bash scripts/download/download_data_all.sh
```

The downloader uses the Hugging Face mirror by default:

```text
https://hf-mirror.com
```

To use the official Hugging Face endpoint:

```bash
HF_ENDPOINT=https://huggingface.co bash scripts/download/download_data_all.sh
```

You can also run the dataset download scripts separately:

```bash
python scripts/download/download_data_modelnet.py
python scripts/download/download_data_modelnet_c.py
python scripts/download/download_data_scanobjectnn.py
python scripts/download/download_data_scanobjectnn_c.py
```

After downloading, the dataset directory should follow this structure:

```text
DPC-Point/
  data/
    modelnet_c/
      shape_names.txt
      clean.h5
      add_global_0.h5
      add_global_1.h5
      ...
      scale_4.h5
    sonn_c/
      shape_names.txt
      hardest/
        clean.h5
        add_global_0.h5
        add_global_1.h5
        ...
        scale_4.h5
```

ModelNet-C and ScanObjectNN-C contain seven corruption types: `add_global`, `add_local`, `dropout_global`, `dropout_local`, `jitter`, `rotate`, and `scale`. The `all35` setting denotes 7 corruption types across 5 severity levels.

### 4. Text Templates and Optional LLM Configuration

Text templates are stored under:

```text
text_templates/
  handcrafted/
    original_handcrafted.json
    pointcloud_depth_view.json
  llm_supplement/
    modelnet_c_deepseek_deepseek-v4-pro_multiview_2d3d_10_prompts.json
    sonn_c_deepseek_deepseek-v4-pro_multiview_2d3d_10_prompts.json
```

The default inference setting uses fused text features from handcrafted templates and LLM-generated class descriptions. If the cached LLM description JSON files already exist, no API key is required for reproduction.

Only when regenerating LLM descriptions, create a `.env` file in the project root:

```text
API_KEY=sk-xxx
BASE_URL=https://api.deepseek.com
MODEL=deepseek-v4-pro
PROVIDER=deepseek
TEMPERATURE=0.3
```

The code loads `.env` with `python-dotenv`. `API_KEY` is used only for online prompt generation and is not written to experiment configuration files.

### 5. Zero-Shot Baselines

Run one backbone on one dataset:

```bash
bash scripts/zero_shot/run_common.sh ulip modelnet_c 0 --severities 2
```

Run each backbone on all four datasets:

```bash
bash scripts/zero_shot/run_ulip.sh 0
bash scripts/zero_shot/run_openshape.sh 0
bash scripts/zero_shot/run_uni3d.sh 0
```

Run all three backbones sequentially:

```bash
bash scripts/zero_shot/run_all.sh 0
```

Here `0` denotes the GPU index. To run on CPU:

```bash
bash scripts/zero_shot/run_common.sh ulip modelnet cpu
```

### 6. DPC-Point Inference

Run one backbone on one dataset:

```bash
bash scripts/dpc_point/run_common.sh uni3d scanobjectnn_c 0 --severity-set s2
```

Run each backbone on all four datasets:

```bash
bash scripts/dpc_point/run_ulip.sh 0
bash scripts/dpc_point/run_openshape.sh 0
bash scripts/dpc_point/run_uni3d.sh 0
```

Reproduce DPC-Point on three backbones and four datasets:

```bash
bash scripts/dpc_point/run_all.sh 0
```

Common settings for corrupted datasets:

```bash
# Run only severity level 2.
bash scripts/dpc_point/run_common.sh uni3d modelnet_c 0 --severity-set s2

# Run 7 corruption types across 5 severity levels.
bash scripts/dpc_point/run_common.sh uni3d modelnet_c 0 --severity-set all35

# Run selected corruption types.
bash scripts/dpc_point/run_common.sh uni3d scanobjectnn_c 0 --severity-set s2 --corruptions rotate,add_global
```

The `scripts/dpc_point/run_*.sh` scripts define the dataset list and the fixed inference setting used for released experiments. Fixed hyperparameters are written to `config.json`; console output only reports the current experiment identity, real-time progress, and OA results.

## Outputs

Zero-shot results are saved to:

```text
results/zero_shot/<experiment_name>/
```

DPC-Point results are saved to:

```text
results/dpc_point/<experiment_name>/
```

Each DPC-Point experiment directory contains:

- `run.log`: full console log
- `summary.csv`: OA results for each evaluated task
- `config.json`: complete experiment configuration, including dataset, backbone, text templates, cache capacities, distribution parameters, and final-score weights
